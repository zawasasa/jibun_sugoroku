「ささの自分すごろく」アプリ開発 要件定義書
Claude Codeに渡すための詳細な要件定義をまとめました。

📋 プロジェクト概要
アプリ名: ささの自分すごろくゲーム
目的: ささの教育実践と日常生活を振り返るすごろくゲームをWebアプリとして実装
技術スタック: HTML/CSS/JavaScript（シンプルなWebアプリ、ブラウザで動作）

🎮 ゲーム仕様
基本ルール

プレイヤー数: 1〜4人（開始時に選択）
ゴール: スタートから60マス目がゴール
勝利条件: 最初にゴールに到達したプレイヤーが勝利
ターン制: プレイヤーが順番にサイコロを振ってコマを進める

マスの種類（3種類）
マスの種類色内容数ポジティブマス🟢 緑止まるとポジティブイベントカードを引く（前進イベント）約55個のイベントからネガティブマス🔴 赤止まるとネガティブイベントカードを引く（後退イベント）約15個のイベントからニュートラルマス🟡 黄止まると選択・ランダムイベントカードを引く約10個のイベントから
マス配置: 60マスをランダムに配置（開始時に生成）

スタートとゴールは固定
ポジティブ: 約30マス
ネガティブ: 約10マス
ニュートラル: 約10マス
通常マス（何も起きない）: 約10マス


🎲 ゲームフロー
1. ゲーム開始
1. トップ画面で「ゲームスタート」ボタン
2. プレイヤー数選択（1〜4人）
3. 各プレイヤーの名前入力（デフォルト: プレイヤー1, 2, 3, 4）
4. プレイヤーごとのコマの色選択（赤・青・緑・黄）
5. ボード生成（マスの配置決定）
6. ゲーム画面へ遷移
2. ターンの流れ
1. 現在のプレイヤーのターン表示
2. 「サイコロを振る」ボタンをクリック
3. サイコロアニメーション（1〜6の目がランダム表示）
4. 出た目の数だけコマが移動（アニメーション）
5. 止まったマスの種類を判定
   - 通常マス → 次のプレイヤーへ
   - イベントマス → イベントカード表示
6. イベントカードの内容表示
7. 「イベント実行」ボタンクリック
8. イベント効果の適用（コマ移動）
9. ゴール判定
   - ゴールしていない → 次のプレイヤーへ
   - ゴール → ゲーム終了画面
3. イベントカード処理
ポジティブイベント:

カードに表示: イベントテキスト + 「◯マス進む」
実行ボタンで前進

ネガティブイベント:

カードに表示: イベントテキスト + 「◯マス戻る」
実行ボタンで後退

ニュートラルイベント:

サイコロ判定型: 「サイコロを振る」ボタン → 結果に応じて進む/戻る
選択型: 「選択肢A」「選択肢B」ボタン → 選択に応じて効果
特殊型: 「1回休み」「ゴール直行」など

## 🔧 実装要件

### 必須機能
1. ✅ プレイヤー設定（1〜4人、名前・色）
2. ✅ すごろくボード生成（60マス、ランダム配置）
3. ✅ サイコロ機能（1〜6のランダム、アニメーション）
4. ✅ コマ移動（出目に応じて移動）
5. ✅ イベント判定（マスの種類に応じてカード表示）
6. ✅ イベント実行（カード内容に応じてコマ移動）
7. ✅ ターン管理（プレイヤー順に切り替え）
8. ✅ ゴール判定・勝利画面

### オプション機能（余裕があれば）
- ⭐ BGM・効果音
- ⭐ コマ移動のアニメーション強化
- ⭐ イベント履歴表示
- ⭐ ゲームログ（誰がどのイベントを引いたか）
- ⭐ セーブ・ロード機能（localStorage使用）

### レスポンシブ対応
- PC（横長画面）
- タブレット
- スマートフォン（縦長画面）
